8 ## Introduction
9 The Button library is for debouncing and reading momentary contact switches like tactile button switches. "Long presses" of arbitrary length can be detected. Works well in state machine constructs. Use the read() function to read each button in the main loop, which should execute as fast as possible.

11 ## Installation
12 To use the **Button** library:
13 - Go to https://github.com/JChristensen/Button, click the **Download ZIP** button and save the ZIP file to a convenient location on your PC.
14 - Uncompress the downloaded file. This will result in a folder containing all the files for the library, that has a name that includes the branch name, usually **Button-master**.
15 - Rename the folder to just **Button**.
16 - Copy the renamed folder to the Arduino sketchbook\libraries folder.

25 ## Button library methods
26 
27 ### Button(pin, puEnable, invert, dbTime)
28 ##### Description
29 The constructor defines a button object.
30 ##### Syntax
31 `Button(pin, puEnable, invert, dbTime);`
32 ##### Parameters
33 **pin:** Arduino pin number that the button is connected to *(byte)*
34 **puEnable:** *true* to enable the microcontroller's internal pull-up resistor, else *false* *(boolean)*
35 invert: *false* interprets a high logic level to mean the button is pressed, *true* interprets a low level as pressed. *true* should be used when a pull-up resistor is employed, *false* for a pull-down resistor. *(boolean)*
36 **dbTime:** Debounce time in milliseconds *(unsigned long)*
37 ##### Returns
38 None.
39 ##### Example
40 ```c++
41 Button myButton = Button(2, true, true, 25); //25 ms debounce
42 ```
43 
44 ### read(void)
45 ##### Description
46 Reads the button and returns a *boolean* value (*true* or *false*) to indicate whether the button is pressed. The read() function needs to execute very frequently in order for the sketch to be responsive. A good place for read() is at the top of loop(). Often, the return value from read() will not be needed if the other functions below are used.
47 ##### Syntax
48 `myButton.read();`
49 ##### Parameters
50 None.
51 ##### Returns
52 *true* if the button is pressed, *else* false *(boolean)*
53 ##### Example
54 ```c++
55 myButton.read();
56 ```

58 ### isPressed(void)
59 ### isReleased(void)
60 ##### Description
61 These methods check the button state at the point in time when it was last read, and return false or true accordingly. These functions **do not** cause the button to be read.
62 ##### Syntax
63 `myButton.isPressed();`
64 `myButton.isReleased();`
65 ##### Parameters
66 None.
67 ##### Returns
68 *true* or *false*, depending on whether the button has been pressed (released) or not *(boolean)*
69 ##### Example
70 ```c++
71 if ( myButton.isPressed() ) {
72  //do some stuff
73 }
74 else {
75  //do some different stuff
76 }
77 ```

79 ### wasPressed(void)
80 ### wasReleased(void)
81 ##### Description
82 These methods check the button state to see if it changed between the last two reads and return false or true accordingly. These functions **do not** cause the button to be read. Note that these functions may be more useful than `isPressed()` and `isReleased()` since they actually detect a **change** in the state of the button, which is usually what we want in order to cause some action.
83 ##### Syntax
84 `myButton.wasPressed();`
85 `myButton.wasReleased();`
86 ##### Parameters
87 None.
88 ##### Returns
89 *true* or *false*, depending on whether the button was pressed (released) or not *(boolean)*
90 ##### Example
91 ```c++
92 if ( myButton.wasPressed() ) { ...
93 ```

95 ### pressedFor(ms)
96 ### releasedFor(ms)
97 ##### Description
98 These methods check to see if the button is pressed (or released), and has been in that state for the specified time in milliseconds. Returns false or true accordingly. These functions are useful to detect "long presses". Note that these functions **do not** cause the button to be read.
99 ##### Syntax
100 `myButton.pressedFor(ms);`
101 `myButton.releasedFor(ms);`
102 ##### Parameters
103 **ms:** The number of milliseconds *(unsigned long)*
104 ##### Returns
105 *true* or *false*, depending on whether the button was pressed (released) for the specified time *(boolean)*
106 ##### Example
107 ```c++
108 if ( myButton.pressedFor(1000) ) { //has the button been pressed for one second?
109 ```

111 ### lastChange(void)
112 ##### Description
113 Under certain circumstances, it may be useful to know when a button last changed state. lastChange() returns the time the button last changed state, in milliseconds (the value is derived from the Arduino millis() function).
114 ##### Syntax
115 `myButton.lastChange();`
116 ##### Parameters
117 None.
118 ##### Returns
119 The time in milliseconds when the button last changed state *(unsigned long)*
120 ##### Example
121 ```c++
122 unsigned long lastChange = myButton.lastChange();
123 ```
